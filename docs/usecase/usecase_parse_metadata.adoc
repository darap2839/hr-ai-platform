= Use Case: Парсинг метаданных резюме
:toc:
:sectnums:

== Акторы

* hr_etl_service
* NLP модели
* hr_ontology_service
* hr_psql_service

== Цель

Извлечь из резюме структуру:
- опыт, - навыки, - образование, - контакты, - должности, - категории навыков.

== Основной сценарий

1. hr_etl_service получает текст резюме.
2. Вызывает NLP pipeline.
3. NLP извлекает сущности (NER).
4. NLP классифицирует навыки через онтологию.
5. Строится JSON модель "CandidateProfile".
6. hr_etl_service сохраняет профиль в базе.

== Альтернативы

* Ошибка классификации → пропуск поля.
* Недоступна онтология → частичная классификация.

== Постусловия

* Профиль кандидата структурирован и готов к сопоставлению.
