== Назначение

Документ служит точкой входа для понимания поведения сервисов и логики взаимодействия в рамках платформы.

Каждый Use Case описывает:
- акторов, - триггеры, - основной сценарий, - альтернативные ветки, - ошибки и исключения, - данные, участвующие в процессе.
